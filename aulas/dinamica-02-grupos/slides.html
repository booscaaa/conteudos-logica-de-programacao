O que vocês podem usar

Somente o que vimos em aula — sem pesquisar estruturas novas!

Liberado

  • ✓ Variáveis: inteiro, real, caracter, logico
  • ✓ leia() e escreva()
  • ✓ Operadores aritméticos e relacionais
  • ✓ Álgebra booleana: e, ou, nao
  • ✓ Estrutura se / senao se / senao
  • ✓ Estrutura escolha / caso / outrocaso
  • ✓ Laço enquanto — com condição de parada
  • ✓ Laço para — contador fixo
  • ! Laço repita / ate — use com cuidado

Proibido

  • ✗ Funções e procedimentos personalizados
  • ✗ Vetores e matrizes

Grupo 1 — Detalhamento

🛒 Caixa de Supermercado

description Briefing do Cliente

O atendente registra produtos pelo código em loop. Digita 0 para encerrar. O programa soma o total e aplica 10% de desconto se passar de R$ 100,00. Produtos 1 a 5 com preços fixos.

Produtos

  • 1 Arroz 5kg — R$ 28,00
  • 2 Feijão 1kg — R$ 9,50
  • 3 Azeite 500ml — R$ 24,90
  • 4 Macarrão 500g — R$ 5,80
  • 5 Molho de Tomate — R$ 4,20

Critérios de aceite

  • ✓ Loop encerra ao digitar código 0
  • ✓ Preço correto somado para cada código
  • ✓ Código inválido → aviso, mas loop continua
  • ! Obrigatório: enquanto + escolha + se
  • ✓ Desconto de 10% apenas acima de R$ 100,00
Grupo 2 — Detalhamento

🎯 Quiz de Conhecimentos

description Briefing do Cliente

Quiz de informática com 3 perguntas de múltipla escolha (A/B/C). Para cada resposta certa soma 1 ponto. No final exibe a medalha: ouro (3), prata (2), bronze (1) ou "tente novamente" (0).

Perguntas e respostas

  • Q1 Operação de resto → resposta: B (%)
  • Q2 Significado de CPU → resposta: A
  • Q3 Operador E no Portugol → resposta: C (e)

Critérios de aceite

  • ✓ 3 perguntas exibidas em sequência pelo para
  • ✓ Ponto somado apenas na resposta certa
  • ✓ Resultado final com medalha correto
  • ! Obrigatório: para + escolha + se
Grupo 3 — Detalhamento

⚽ Rodadas do Campeonato

description Briefing do Cliente

Registra 5 rodadas de futebol com gols do time A e B. Vitória = +3 pts, empate = +1 pt, derrota = 0. No final classifica: campeão (≥12), playoff (7–11) ou rebaixamento (<7).

Pontuação por rodada

  • +3 Vitória — gols A > gols B
  • +1 Empate — gols A == gols B
  • +0 Derrota — gols A < gols B

Critérios de aceite

  • ✓ Exatamente 5 rodadas pedidas em sequência
  • ✓ Acumulador de pontos correto
  • ✓ Classificação final correta para cada faixa
  • ! Obrigatório: para + se/senão + acumulador
Grupo 4 — Detalhamento

💼 Comissão do Vendedor

description Briefing do Cliente

Vendedor registra cada venda pelo tipo (1 a 4). Digita 0 para encerrar. O programa acumula a comissão de cada tipo. Se o total ultrapassar R$ 500,00, aplica bônus de 20%.

Comissão por tipo

  • 1 Eletrônico — R$ 50,00
  • 2 Eletrodoméstico — R$ 35,00
  • 3 Roupa — R$ 15,00
  • 4 Acessório — R$ 8,00

Critérios de aceite

  • ✓ Loop encerra ao digitar tipo 0
  • ✓ Comissão correta somada por tipo
  • ✓ Bônus de 20% apenas quando total > R$ 500
  • ! Obrigatório: enquanto + escolha + se
Grupo 5 — Detalhamento

🔐 Sistema de Acesso

description Briefing do Cliente

O usuário tem até 3 tentativas para digitar o PIN correto (1234). Se acertar, acessa o menu com 3 opções. Se errar 3 vezes, o sistema bloqueia o acesso.

Regras

  • ✓ PIN 1234 correto → acesso liberado
  • ! PIN errado → incrementa tentativas
  • ✗ 3 erros → bloqueio definitivo
  • 1 Menu: Ver relatório
  • 2 Menu: Registrar ponto
  • 3 Menu: Sair

Critérios de aceite

  • ✓ Loop para imediatamente ao acertar o PIN
  • ✓ Exatamente 3 tentativas antes de bloquear
  • ✓ Menu aparece apenas se autenticado
  • ! Obrigatório: enquanto + se/senão + escolha

Para o Dev

Dicas de Código

Padrão ENQUANTO — acumulador (Desafios 1, 4 e 5)

inteiro codigo real total total = 0.0 escreva("Código (0 p/ sair): ") leia(codigo) enquanto (codigo != 0) { escolha(codigo) { caso 1: total = total + 28.0 outrocaso: escreva("Inválido") } escreva("Código (0 p/ sair): ") leia(codigo) // relê DENTRO do loop! } se (total > 100.0) { total = total * 0.90 }

Padrão PARA — contador fixo (Desafios 2 e 3)

inteiro i, pontos caracter resp pontos = 0 para i = 1 ate 3 faca { escolha(i) { caso 1: escreva("Qual o resto? A)/ B)% C)*") leia(resp) se (resp == 'B') { pontos = pontos + 1 escreva("Certo! +1 ponto") } senao { escreva("Errado. Era B") } } } // pontos: 0..3 → usa escolha(pontos)
warning No enquanto: releia a variável de controle no final do loop, senão ele nunca para!
Para o QA

Como montar seus Testes

Crie a tabela de testes antes mesmo do código ficar pronto — é trabalho em paralelo!

# Entradas (em ordem) Saída esperada Resultado
CT01Caso feliz — entrada válida que encerra corretamenteSaída e totais corretos— a testar
CT02Outro valor válido diferenteResposta correta esperada— a testar
CT03Valor limite que aciona regra de bônus/desconto/prêmioBônus/desconto aplicado— a testar
CT04Encerrar imediatamente (0 ou loop não executado)Total zero, sem bônus— a testar
CT05Entrada inválida (código/tipo fora do esperado)Mensagem de erro amigável— a testar
CT06Caso extremo — mínimo ou máximo possívelTratamento correto do limite— a testar
info Após rodar: preencha cada linha com OK ou FALHA + o que saiu errado. Verifique especialmente se o loop para na hora certa.
Para o UX Writer

Textos que fazem diferença

Textos ruins

  • ✗ "Digite opcao:"
  • ✗ "Erro."
  • ✗ "ok"
  • ✗ "total = 87.5"
  • ✗ Sem saudação ao iniciar
  • ✗ Sem mensagem de encerramento do loop

Textos bons

  • ✓ "Bem-vindo ao Caixa do MercadoBom!"
  • ✓ "Código do produto (0 para sair): "
  • ✓ "Arroz 5kg adicionado. +R$ 28,00"
  • ✓ "Desconto de 10% aplicado! Economizou R$ X."
  • ✓ "Total a pagar: R$ X. Obrigado!"
  • ✓ "PIN incorreto. Tentativas restantes: 2."
