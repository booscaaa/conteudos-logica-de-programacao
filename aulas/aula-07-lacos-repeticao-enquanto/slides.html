Comparação

para vs enquanto

123
para
Use quando você sabe exatamente quantas vezes repetir.

Exemplos:
✓ Imprimir 10 números
✓ Calcular tabuada até 10
✓ Ler notas de 30 alunos
para(inteiro i = 1; i <= 10; i++) {
  escreva(i)
}
autorenew
enquanto
Use quando você não sabe quantas vezes repetir — depende de uma condição.

Exemplos:
✓ Pedir senha até acertar
✓ Somar até digitar 0
✓ Jogar até perder
enquanto (i <= 10) {
  escreva(i)
  i++
}
💡 Dica prática: se você precisaria colocar "até que o usuário decida parar", use o enquanto. Se é "repita N vezes", use o para.
Estrutura

Anatomia do enquanto

enquanto (condição) {
    // bloco executado enquanto a condição for verdadeira
    // IMPORTANTE: atualizar a condição aqui!
}
❓ Condição
Uma expressão lógica testada antes de cada execução. Se for verdadeira, o bloco roda. Se falsa, para.
▶ Bloco de código
Tudo entre { e } é executado repetidamente enquanto a condição for verdadeira.
⚡ Atualização
Você precisa mudar algo dentro do bloco para que, em algum momento, a condição se torne falsa.
warning  Diferente do para, o enquanto não tem inicialização nem incremento automático — você declara as variáveis antes e as atualiza dentro do bloco!
Fluxo de Execução

Como o enquanto funciona?

Passo a passo
1
Testa a condição
O programa avalia se a expressão dentro dos parênteses é verdadeira ou falsa.
2
Se verdadeira → executa o bloco
Todo o código entre { e } é executado uma vez (uma iteração).
3
Volta ao passo 1
A condição é testada novamente. Se ainda verdadeira, executa outra vez.
4
Se falsa → encerra o laço
O programa sai do bloco e continua executando o que vem depois.
programa { funcao inicio() { inteiro i = 1 // 1. declara antes enquanto (i <= 5) { // 2. testa escreva(i) // 3. executa i = i + 1 // 4. atualiza! } // volta ao 2 } }
🔑 A variável i é declarada fora do enquanto, e atualizada dentro.

Padrão Clássico

Validação de entrada

programa { funcao inicio() { inteiro nota = -1 // valor inválido inicial enquanto (nota < 0 ou nota > 10) { escreva("Digite nota (0-10): ") leia(nota) } escreva("Nota válida: ", nota) } }
🎯 Inicialize a variável com um valor inválido (-1) para garantir que o laço execute ao menos uma vez na primeira rodada!

Atenção!

A armadilha do loop infinito

dangerous
Loop infinito — ERRADO
inteiro i = 1
enquanto (i <= 5) {
  escreva(i)
  // ← i nunca muda!
}
i = 1 para sempre → condição nunca fica falsa → programa trava!
check_circle
Correto — com atualização
inteiro i = 1
enquanto (i <= 5) {
  escreva(i)
  i = i + 1 // ← i cresce!
}
i aumenta a cada volta → chega a 6 → condição falsa → encerra ✓
warning  Regra de ouro: sempre garanta que algo dentro do bloco altere a condição, caso contrário o programa nunca sai do laço!
info  Também tome cuidado com condições usando leia() — o usuário pode não digitar o valor esperado nunca. Sempre informe na tela o que ele deve digitar para sair.
