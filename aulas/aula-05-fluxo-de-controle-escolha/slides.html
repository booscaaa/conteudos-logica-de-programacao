Contexto

Você já conhece o SE

Até aqui, usamos se / senao se / senao para tomar decisões. Mas existe um problema:

1

Quando testamos muitas opções do mesmo valor, o código fica longo e difícil de ler.

2

Repetimos a variável em cada condição: se (dia == 1)... se (dia == 2)... se (dia == 3)...

3

O código perde clareza e fica difícil de manter.

lightbulb A estrutura escolha resolve exatamente esse problema!
Conceito

O que é a Estrutura Escolha?

switch
em C, Java, JS
→
escolha
em Portugol

É uma forma elegante e eficiente de verificar o valor de uma variável contra várias opções possíveis.
Funciona como um cardápio de decisões.

restaurant_menu Ideal quando você sabe todos os valores possíveis antecipadamente.
Sintaxe

A Anatomia do Escolha

Componentes

Os 4 Componentes Fundamentais

escolha (variavel)
1. A palavra-chave
Define qual variável será analisada. O computador avaliará essa variável contra cada caso.
caso valor:
2. Cada caso
Define um valor possível. Se a variável for igual a esse valor, o bloco abaixo é executado.
pare
3. O pare — CRÍTICO!
Encerra a execução do bloco. Sem ele, o código continua executando os casos seguintes!
caso contrario:
4. O caso contrário
Opcional. É executado quando nenhum dos casos anteriores foi satisfeito. Equivale ao senao.

O Perigo do Fall-Through

O Pare é Obrigatório!

✗ SEM pare — ERRADO

escolha (nota) { caso 10: escreva("Excelente!") // Sem pare! Cai no próximo caso 9: escreva("Muito bom!") pare }
Se nota=10: escreve "Excelente E "Muito bom — bug!

✓ COM pare — CORRETO

escolha (nota) { caso 10: escreva("Excelente!") pare // Para aqui! caso 9: escreva("Muito bom!") pare }
Se nota=10: escreve apenas "Excelente!" e para.

Comparando Código

O mesmo código — dois jeitos

Com se / senao se

se (dia == 1) { escreva("Domingo") } senao se (dia == 2) { escreva("Segunda") } senao se (dia == 3) { escreva("Terça") } senao se (dia == 4) { escreva("Quarta") } senao { escreva("Inválido") }

Com escolha / caso

escolha (dia) { caso 1: escreva("Domingo") pare caso 2: escreva("Segunda") pare caso contrario: escreva("Inválido") }
💡 O escolha é mais legível, menos repetitivo e deixa a intenção do código mais clara.

Técnica Avançada

Agrupando Casos

Múltiplos casos podem compartilhar o mesmo bloco de código — basta listá-los em sequência, sem pare entre eles.

escolha (mes) { caso 1: caso 3: caso 5: caso 7: caso 8: caso 10: caso 12: escreva("31 dias") pare caso 4: caso 6: caso 9: caso 11: escreva("30 dias") pare caso contrario: escreva("Fevereiro ou inválido") }
💡 Meses de 31 dias compartilham o mesmo bloco. Elegante e sem repetição!

Boas Práticas

Dicas Importantes

1

Sempre inclua o caso contrario

Mesmo que você pense que nunca será ativado, ele protege o programa de entradas inesperadas.

2

Nunca esqueça o pare

O fall-through é um dos bugs mais comuns e difíceis de identificar.

3

Prefira escolha para valores discretos e conhecidos

Quando o conjunto de valores é finito e previsível (dias da semana, meses, opções de menu), o escolha é mais claro que o se.

4

Mantenha os casos em ordem lógica

Organize os casos numericamente ou por frequência de uso. Coloque o caso contrario sempre por último.
